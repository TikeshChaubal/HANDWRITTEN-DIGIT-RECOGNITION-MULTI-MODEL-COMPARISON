HANDWRITTEN DIGIT RECOGNITION — MULTI-MODEL COMPARISON

ABSTRACT
Handwritten digit recognition is a classic supervised learning problem in the field of computer vision. This project applies and compares six machine-learning classifiers — Random Forest, Logistic Regression, Linear SVM, Non-Linear SVM (RBF kernel), XGBoost, and Decision Tree — on the MNIST dataset formatted for the Kaggle competition "Digit Recognizer". Each model is trained on 28×28 grayscale pixel values (flattened to 784 features) and evaluated using accuracy, per-class classification reports, ROC curves (One-vs-Rest), and a side-by-side bar chart comparison. The best-performing model is identified automatically and its predictions are exported as a submission CSV. """

OBJECTIVE
1. Load and explore the MNIST training and test datasets.
2. Preprocess the data (normalisation, train/validation split).
3. Train six classifiers with consistent settings.
4. Compare models using validation accuracy, classification reports, and One-vs-Rest ROC-AUC curves.
5. Visualise accuracy comparisons and ROC curves side-by-side.
6. Perform predictive testing on sample validation images.
7. Generate a submission-ready CSV for Kaggle.
8. State conclusions based on the empirical results.

Conclusion
This project looked at six machine learning classifiers to see how well they could recognize digits in the MNIST dataset. The classifiers were Random Forest, Logistic Regression, Linear SVM, Non-Linear SVM, XGBoost and Decision Tree.

The results showed that the classifiers that were not linear and the ones that used models together did a lot better than the simple linear models. Non-Linear SVM and XGBoost were the best they got it right 97 to 98 percent of the time. These two models were very good at telling the difference between the ten digit classes they were almost perfect.

Random Forest and Logistic Regression were not as good. They were still very good and they did not take as long to train. Decision Tree was the worst it had a time with the big dataset.

The model that was chosen worked well on the test dataset and it saved the answers to a file called submission.csv so it could be uploaded to Kaggle. These machine learning methods are good for computer vision. If you want to get even better results you should use Convolutional Neural Networks. They can get it more, than 99 percent of the time.
